# Syntrix Demo Skill

Use this skill when improving the Syntrix local hackathon demo.

## Goal

Keep Syntrix positioned as a serious reasoning-agent product that discovers personalized agent opportunities from synthetic Copilot-style work patterns.

## Rules

- Preserve the synthetic-data-only boundary.
- Do not add real secrets or paid API requirements.
- Favor clear product storytelling over abstract agent jargon.
- Keep Streamlit interactions polished and easy to demo.
- Keep generated blueprints safe, auditable, and human-approval aware.

## Required Labels

- Syntrix Reasoning Engine
- Syntrix Opportunity Score
- Syntrix Agent Blueprint
- Syntrix Continuous Improvement Loop

## Demo Narrative

1. Select a role in the sidebar.
2. Show repeated synthetic Copilot-style interactions.
3. Explain how Syntrix finds repeated workflows.
4. Show the Syntrix Opportunity Score chart.
5. Open the top recommended agent opportunity.
6. Present the generated Syntrix Agent Blueprint.
7. Close with safety review and continuous improvement.
