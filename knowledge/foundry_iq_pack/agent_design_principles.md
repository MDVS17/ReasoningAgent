# Agent Design Principles

## Purpose

This synthetic source defines design principles for Syntrix Agent Blueprints. It is a local demo grounding document and does not contain real company data.

## Principles

- Start from repeated work signals, not abstract automation ideas.
- Define a narrow purpose, target user, triggering patterns, and expected outputs.
- Keep the agent advisory until human approval is granted for sensitive or write-capable actions.
- Include evaluation tests before recommending deployment.
- Make assumptions, source context, and confidence visible to the user.

## Blueprint Guidance

A high-quality agent blueprint should explain why the agent is needed, what signals triggered the recommendation, what tools it may use, and which approval gates are required.
